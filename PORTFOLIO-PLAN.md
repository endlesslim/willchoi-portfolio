## 1. 현재 상태 진단

### 이미 잘 되어 있는 것 (건드리지 말 것)
- **포지셔닝 문장**: "써본 사람 말고, 팔아본 사람이 만드는 디자인" — 이대로 유지. AI 시대에 가장 방어력 높은 프레임.
- **팝캔티슈 리오더 케이스 스터디**: 문제 → 리서치 → 페르소나 → 플로우 → UI → 검증 구조가 완성형.
- **아카이브 4종**(카카오톡 GUI 2011, IONOI 2011, empal 2005, Cooking Atelier 2016): 연차 증명용으로 적절.
- **타임라인 + About**: 사업 운영 경험을 앞세운 서사 방향이 맞음.

### 부족한 것 3가지
1. **케이스 스터디가 1개뿐** — 보는 사람은 "이 사람의 실력이 우연인지 패턴인지"를 확인하려고 최소 2~3개를 본다.
2. **AI 활용 역량이 어디에도 안 보임** — 실제로는 AI 파이프라인으로 서비스를 혼자 출시하는 수준인데, 포트폴리오에는 흔적이 없다. 지금 시장에서 가장 비싸게 팔리는 스펙이 숨겨져 있는 상태.
3. **타임라인 공백(2012~2024)** — 비워두면 보는 사람이 상상으로 채운다. 한 줄이라도 채워야 함.

## 2. 목표 구조 — 케이스 스터디 4개의 스토리라인

포트폴리오 전체가 하나의 문장이 되게 한다:

> **"브랜드를 만들고 → 화면으로 옮기고 → AI와 함께 혼자 출시까지 한다."**

| # | 케이스 | 증명하는 능력 | 자산 상태 |
|---|--------|--------------|----------|
| 1 | 팝캔티슈 리오더 앱 (완성) | UX 리서치·UI 디자인 | ✅ 완성 |
| 2 | 팝캔티슈 브랜드 & 패키지 | 브랜드 아이덴티티·실물 제품화 | ✅ 자산 보유 (캐릭터 6종 PNG, 전개도, 실제 제품 사진, 라이프스타일 목업 — assets/ 폴더에 이미 있음) |
| 3 | 타고(사주 궁합 앱) | AI-native 프로덕트 디자인·실서비스 운영 | ✅ 실서비스 운영 중 (saju-v2) |
| 4 | 봉오르디 100대명산 앱 | 접근성 퍼스트 디자인 시스템 (50대+) | ✅ 앱 완성 + DESIGN.md 디자인 시스템 문서 보유 |

### 각 케이스의 핵심 앵글

**② 팝캔티슈 브랜드 & 패키지 — "화면이 아니라 공장에서 완성되는 디자인"**
- 캐릭터 6종 기획 → 패키지 전개도 → 인쇄·제조 → 실제 매장 입점까지 전 과정.
- 다른 디자이너가 흉내 못 내는 지점: 인쇄 도수, 원가, 전개도 같은 제조 제약 안에서 디자인한 경험.
- 리오더 앱 케이스와 짝을 이룸 (같은 브랜드의 실물→디지털).

**③ 타고 — "디자이너 혼자, AI와 함께, 기획부터 출시까지"**
- 관계·궁합 소셜 앱. 무가입 온보딩(전환율 설계), AI 채팅 UX, 실사용자 리뷰 수집까지 실제 운영.
- 핵심 메시지: 화면 제작은 AI에게, 무엇을 만들지·무엇이 좋은 디자인인지 판단은 나에게.
- "AI 때문에 디자이너가 위험하다"는 프레임을 "AI 덕분에 디자이너 혼자 팀 하나 몫을 한다"로 뒤집는 케이스.

**④ 봉오르디 앱 — "50대+를 위한 접근성 퍼스트 디자인 시스템"**
- 루트 폰트 17px, 버튼 최소 높이 52px, WCAG AA 대비 보장 등 구체 수치가 이미 DESIGN.md에 정리돼 있음 → 그대로 케이스 스터디 소재.
- 시니어 타겟 UX는 명확한 니치 — "누구를 위해 왜 이렇게 정했는가"를 보여주기 좋은 프로젝트.

## 3. 신규 섹션: "How I Work" (AI 워크플로)

index.html의 About 위나 아래에 짧은 섹션 하나 추가.

- **말할 것**: 제작은 AI 도구로 몇 배 빠르게, 방향 설정·품질 판단·브랜드 일관성은 직접. 실제로 PM→디자인→개발→QA 파이프라인을 직접 구축해 운영 중.
- **증거**: 타고(실서비스 출시), CrossVault 3PL 영업 데모(기획→배포 전 과정), 콘텐츠 제작 파이프라인.
- **톤 주의**: "AI 잘 씀"이 아니라 "AI에게 정확히 지시하고 결과물을 골라내는 안목" 쪽으로. 도구 나열은 하지 않는다.

## 5. 정직성 점검 (중요)

리오더 케이스의 수치("거래처 6곳 인터뷰", "5/6 성공, 평균 38초")가 **실제 수행한 검증인지 확인할 것.**
- 실제라면 그대로 두면 됨 (가장 강한 무기).
- 콘셉트용 가상 수치라면, 면접·미팅에서 "그 테스트 어떻게 하셨어요?" 한 방에 무너진다. "검증 계획" 수준으로 표현을 낮추거나, 실제로 거래처 몇 곳에 프로토타입을 보여주고 진짜 데이터로 교체할 것. 실제 거래처가 있으니 후자가 어렵지 않고 훨씬 강하다.

## 6-1. 대기업 지원용 편집본 스펙 (지원할 일이 생기면 그때 제작)

> 2026-07-10 확정. 현재 라이브 버전은 프리랜서·수주 시장용으로 유지하고,
> 대기업 지원 시점에 아래 스펙으로 별도 편집본을 만든다.

**왜 별도인가:** 대기업 채용은 직무 적합성(role fit)과 협업 신호를 본다. "혼자 다 합니다 + 사업 운영"은 제너럴리스트/이탈 리스크로 읽힐 수 있음.

**바꿀 것:**
- **헤드라인**: "만들고, 출시하고, 팝니다" → "비즈니스 성과로 증명하는 디자인" 계열로 순화
- **정체성 프레임**: 사업가 → "비즈니스를 이해하는 시니어 프로덕트 디자이너". 비라잇 운영은 '사업'이 아니라 '비즈니스 임팩트를 직접 검증한 경험'으로 서술
- **케이스 서술에 협업·프로세스 추가**: 제조 파트너(OEM 인쇄소)와의 협업, 거래처 조율, 품질 기준·일정 관리, 이해관계자 의사결정 — "혼자"보다 "함께 + 책임"
- **AI 활용**: "혼자 팀 몫" → "AI로 팀의 속도를 높이는 방법을 검증한 경험"으로 조직 언어화
- **지표 유지**: 쿠팡 1위, 리뷰 3,000+, 4.6점은 그대로 (대기업에서도 가장 강한 무기)
- **산출물 형태**: 웹 편집본 + **PDF 포트폴리오** (대기업 서류는 PDF 요구가 많음)

**바꾸지 않을 것:** 케이스 구조, 실서비스 링크, 실물 증거 사진, 다크 테크니컬 디자인 시스템.

**면접 단계 대비 메모**: 겸업/이탈 우려 질문에 대한 답변 준비 (브랜드 운영은 유지·위임 가능한 구조라는 점, 조직에서 하고 싶은 일이 무엇인지).

## 7. 디자인 시스템 (2026-07-09 리디자인)

네이비 B2B SaaS 스타일 → **"종이와 잉크(Paper & Ink)" 에디토리얼**로 전면 교체.

- **배경**: 따뜻한 종이색 (#F7F2E9) + 은은한 종이 질감(노이즈) 오버레이
- **글자**: 웜 잉크 (#221D15), 본문 웜 브라운 (#4B4335)
- **포인트 컬러 1개**: 도장 주홍 (#BC4A2A) — 강조 단어, 숫자, 링크 밑줄에만 사용
- **타이포**: 제목 = 고운바탕(세리프), 본문 = Pretendard. 히어로의 "팔아본 사람"이 세리프+주홍으로 강조됨
- **원칙**: 그림자 대신 얇은 웜 헤어라인 보더, 필 태그 제거, 카드 라운드 축소(10~12px), 푸터·CTA는 딥 브라운
- 이후 추가되는 케이스 스터디 페이지도 이 시스템을 따를 것 (style.css의 토큰만 쓰면 자동 적용)

## 8. 포트폴리오 "밖"에서 필요한 것 (사이트 완성과 별개)

1. **배포 갱신 + 도메인** — 이미 GitHub Pages에 배포돼 있음 (github.com/endlesslim/willchoi-portfolio). 리디자인·신규 케이스를 반영하려면 git push 필요. 커스텀 도메인(예: willchoi.kr, 연 2만원 내외)을 붙이면 명함·이메일 서명에 쓸 주소가 생김.
2. **프로필 사진** — 현재 About의 "WC" 이니셜 대신 실제 얼굴 사진. 신뢰도에 가장 싸고 큰 투자.
3. **원페이지 PDF** — 미팅·지원 시 첨부할 한 장짜리 요약 (포지셔닝 한 줄 + 케이스 3개 썸네일 + 연락처). 포트폴리오와 같은 디자인 시스템으로.
4. **채널 정리** — LinkedIn 프로필을 사이트와 같은 포지셔닝 문장으로 통일. 케이스 스터디를 브런치·LinkedIn에 글로 재발행하면 포트폴리오로 들어오는 유입 경로가 생김 (사이트는 목적지일 뿐, 유입은 글이 만든다).
5. **추천사 2~3개** — 실제 거래처 사장님·클라이언트의 짧은 코멘트. AI 시대에 "진짜 사람과 일해본 증거"만큼 강한 게 없음. 받으면 index.html에 섹션 추가.
6. **연락 창구 점검** — 현재 개인 휴대폰 번호가 사이트에 공개돼 있음. 스팸 수신 감수할지, 이메일만 남길지 판단 필요.
