## 1. 제품 정의 — 이 포트폴리오는 무엇인가

포트폴리오도 하나의 "전환 페이지"다. 아래 네 줄이 모든 판단의 기준.

| 항목 | 정의 |
|------|------|
| **누구를 위한 것** | ① 웹/브랜드 디자인을 맡길 소상공인·스타트업 대표 (수주) ② 협업·외주 제안자 ③ (조건부) 채용 담당자 — §8 편집본으로 대응 |
| **주 혜택** | "화면만 그리는 디자이너가 아니라, **팔리는 것까지 증명한 디자이너**를 쓸 수 있다" |
| **부 혜택** | AI 파이프라인으로 혼자 기획→디자인→개발→출시까지 하므로 속도·비용이 팀 단위보다 유리 |
| **Moat (남이 흉내 못 내는 것)** | 쿠팡 카테고리 1위 실물 판매 + 라이브 서비스 운영 + 제조 제약(도수·원가·전개도) 안에서의 디자인 경험 |

**한 문장 스토리라인 (전 케이스가 이 문장을 증명):**
> "브랜드를 만들고 → 화면으로 옮기고 → AI와 함께 혼자 출시까지 한다."

**목표 행동 (전환):** 방문자 기준 ①케이스 스터디 1개 이상 정독 → ②메일 클릭. 사이트의 모든 요소는 이 두 행동으로 수렴해야 하며, 기여하지 않는 요소는 뺀다.

## 2. 방문자 여정 — 섹션별 역할과 답해야 할 질문

각 섹션은 방문자의 질문 하나에 답한다. (Attract → Intrigue → Deliver)

| 순서 | 섹션 | 방문자의 질문 | 답 (핵심 메시지) | 증거 |
|------|------|--------------|----------------|------|
| 1 | 히어로 | "누구지?" | 만들고, 출시하고, 판다 | 실물 제품 사진 + 캡션 "지금 매장에서 팔리는 실물" |
| 2 | 증거 바 | "진짜야?" | 숫자로 즉답 | 쿠팡 1위 · 리뷰 3,000+ · 라이브 3개 |
| 3 | Work | "뭘 했는데?" | 실브랜드의 진짜 문제를 끝까지 | 케이스 2개(팝캔·타고) + Also Shipping |
| 4 | About | "어떤 사람이지?" | 디자인을 사업으로 팔아본 사람 | 프로필 카드 + 타임라인 + 실물 증거(전개도→실물) |
| 5 | (신규) How I Work | "요즘 방식으로 일하나?" | AI에게 제작을, 판단은 나에게 | 타고 출시·파이프라인 운영 사실 |
| 6 | Footer | "연락해볼까?" | 낮은 문턱의 연락처 | 메일·전화 |

## 4. 남은 작업 실행 스펙

### 4-1. "How I Work" 섹션 (index, About 아래)

- **구성 (3단)**: ①원칙 한 문장 — "제작은 AI에게, 판단은 나에게" ②실제 파이프라인 — 기획→디자인→개발→QA를 AI 에이전트로 직접 구축·운영 ③증거 링크 — 타고(기획부터 출시까지 혼자), 셀포트(기획→배포)
- **금지**: 도구 이름 나열(Midjourney·Claude 등 로고 벽), "AI 전문가" 류 자기 수식. 시장에서 흔해빠진 표현이라 오히려 신뢰를 깎는다.
- **형태**: 기존 `next-list`(번호 스택) 재사용, 새 컴포넌트 만들지 않음.
- **완료 기준**: 3개 항목, 각 2문장 이내, 증거 링크 2개 이상, 모바일 확인.

### 4-2. 봉오르디 케이스 스터디 (`case-bongordee.html`)

- **앵글**: "50대+를 위한 접근성 퍼스트 디자인 시스템" — 니치가 명확해 차별화 강함.
- **구조 (case-popcan과 동일 템플릿)**: 히어로(메타 바: 역할/타깃/상태/플랫폼) → 문제(시니어에게 기존 등산앱이 왜 안 되는가 3가지) → 결정(루트 폰트 17px, 버튼 최소 52px, WCAG AA 대비 — **DESIGN.md의 실제 수치 그대로**) → 실물 연결(여권·지도·배지 20종 = 실물과 앱이 한 브랜드) → 다음 단계.
- **자산**: bongordee 저장소 DESIGN.md, 앱 스크린샷, 실물 제품 사진(assets/projects/에 일부 보유).
- **완료 기준**: 구체 수치 5개 이상 인용, 실물↔앱 연결 시각화 1개, index Work 그리드에 카드 승격(현재 Also Shipping에서 이동).

### 4-3. 추천사 (Testimonial)

- **누구에게**: 팝캔 실거래처 사장님 1~2명 + 외주/협업 경험자 1명.
- **요청 템플릿 (카톡용)**: "포트폴리오에 거래처 한마디를 싣고 싶습니다. '어떤 점 때문에 계속 거래하는지' 한두 문장이면 충분합니다. 실명(또는 상호)으로 실어도 될까요?"
- **게시 규칙 (가이드 원칙)**: 실명·실상호 필수, 문장 다듬기 금지(너무 매끈하면 가짜로 보임), 3개 이하.
- **완료 기준**: 실명 코멘트 2개 이상 확보 시에만 섹션 추가. 그 전엔 만들지 않는다.

### 4-4. 방문 측정

- 목표 행동(케이스 클릭, 메일 클릭)이 실제 일어나는지 봐야 다음 수정에 근거가 생긴다.
- **도구**: GoatCounter(무료·무쿠키·스크립트 1줄) 우선. GA는 과함.
- **볼 지표 3개만**: 방문수 / 케이스 페이지 도달률 / mailto 클릭. 월 1회 확인.

### 4-5. 배포·도메인

- 반영 대기 중인 변경: 히어로 v2, 증거 바, About 프로필 카드, 접근성 보강 → **git push 필요 (오너 승인 대기)**.
- 커스텀 도메인(연 2만원 내외): 명함·서명용. 후보 확보만 해두고 급하지 않음.

## 5. 콘텐츠 원칙 (모든 신규 콘텐츠 공통 — 가이드 요약)

1. **구체 숫자 우선.** "많은 리뷰" 금지 → "리뷰 3,000+". 단, **실측 아닌 수치 게시 금지** — 면접·미팅 질문 한 방에 무너진다.
2. **과장 형용사 금지.** 완벽/최고/혁신/seamless 류. 결과(1위, 3,000개)가 말하게 한다.
3. **비주얼은 카피를 증명하는 것만.** 예쁘기만 한 장식 이미지는 넣지 않는다. (실물 사진 > 목업 > 추상 그래픽)
4. **CTA는 화면당 1개.** 문구는 다음 행동을 명시("케이스 스터디 보기", "팝캔티슈 홈페이지 열어보기").
5. **모든 요소에 "왜"가 있어야 한다.** 설명 못 하는 요소는 뺀다.

## 7. 디자인 시스템 — 현행 (다크 테크니컬) ※ v1의 "종이와 잉크"는 폐기됨

- **배경**: 니어블랙 `#0A0A0B` + 필름 그레인 오버레이 / 표면 `#101012`·`#121214`
- **포인트 1개**: 애시드 그린 `#D3F04B` — 강조 단어·라벨·숫자·CTA 호버에만 (화면의 ~5%)
- **타이포**: Pretendard Variable(본문·헤드라인) + IBM Plex Mono(유틸 라벨·숫자, uppercase+letter-spacing). 세리프(고운바탕)는 케이스 내 폰 목업 등 제한 사용
- **헤드라인**: 클수록 타이트 — 히어로 88px/lh 1.06/-.035em
- **구조 언어**: 헤어라인 보더(`--line`), 라운드 3~10px, 필 태그 금지, 모노 라벨 + 값 패턴(증거 바·메타 바·프로필 카드 공통)
- **리듬**: 섹션 140/90/72px, CTA 높이 ≈48px
- **신규 페이지 규칙**: style.css 토큰만 사용. 새 색·새 컴포넌트 추가 전에 기존 패턴(meta-bar, next-list, issue-list, proof-grid) 재사용 검토
